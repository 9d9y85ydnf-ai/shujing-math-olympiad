# 数学竞赛项目工作规范

## 1. 总原则

本目录中的任务默认属于中国高中数学竞赛。解题目标不是只得到答案，而是给出可检查、可学习、可迁移的证明。

正式成品默认采用以下顺序：

1. 题目与条件整理；
2. `解`：可直接誊写的主证明；
3. 等号成立条件或取值说明；
4. `思路`、`点评`或`推广`：只在确有帮助时补充。

## 2. 知识边界

- 默认只使用高中数学竞赛常见知识：代数恒等变形、基本不等式、柯西不等式、排序不等式、切比雪夫不等式、归纳法、函数基本性质、三元对称式和 (pqr) 方法等。
- 不把微积分、拉格朗日乘子、凸分析或计算机搜索作为默认成品证明。
- 若题目结构确实适合切线法，优先给出代数形式的支撑线、因式分解或恒非负证明。
- 若需要展示高等方法，只能作为“拓展”或“内部检验”，并明确说明其知识边界。

## 3. 审题检查

正式推导前必须确认：

- 变量的定义域、正负性和是否允许为零；
- 分母、开方、对数等表达式的合法性；
- 题目是求最值、证明不等式、求参数范围还是证明存在性；
- 表达式是对称、循环还是一般形式；
- 是否齐次，能否归一化；
- 等号可能出现的位置，以及各步放缩的等号能否同时成立。

## 4. 代数题的默认方法顺序

不强行套用固定顺序，但通常优先检查：

1. 齐次化、归一化与换元；
2. 基本不等式、均值不等式和局部放缩；
3. 柯西不等式及其 Engel 形式；
4. 排序不等式、切比雪夫不等式与重排；
5. 调整法：保持关键约束不变，把变量推向边界或等值点；
6. 琴生/切线法：构造统一的线性下界或上界；
7. 三元对称式的 (p=x+y+z, q=xy+yz+zx, r=xyz) 降维；
8. 归纳法、递推法和张瑞祥法等结构性方法。

方法选择必须服务于题目结构。不得为了使用某个技巧而制造无关的复杂变形。

## 5. 证明书写要求

- 关键等价变形写出必要条件，不能把非等价的放缩写成等号。
- 使用柯西、均值、排序等定理时，给出本题中的具体代入形式。
- “显然”“易得”“由对称性”只能用于真正短且可复核的步骤；关键步骤必须展开。
- 证明最值时同时给出上界/下界和达到该值的取值。
- 证明不等式时必须检查等号链，而不是只检查最后一行。
- 变量循环使用时明确下标约定，避免把循环式误当成对称式。

## 6. 题解文件规范

每道题单独保存为 Markdown 文件，建议文件名格式为：

```text
来源-年份-主题.md
```

正文使用 `templates/代数题解模板.md`。不把未经核验的草稿、截图或临时计算直接放入正式题解目录。

## 7. GitHub 发布规范

- 发布前先查看 `git status` 和 `git diff`。
- 只提交本次任务相关文件，不静默包含其他改动。
- 提交信息使用简短、明确的动作描述，例如 `add algebra inequality solution`。
- 首次发布目标为公开仓库 `shujing-math-olympiad`，默认分支为 `main`。
- 后续更新保持目录结构稳定；规范变化、方法新增和题解新增都记录在 `CHANGELOG.md`。
